## Session History & Continuation Guide
*Updated: 2026-03-18*

### What Has Been Built

**Hardware target:** ESP32 WROOM-32 + 2× MKS Servo42C (CR_UART mode) + Xbox One S via Bluepad32 Bluetooth.

**Firmware stack (PlatformIO / Arduino-ESP32):**
- `src/drivers/MKSServo42C.h/.cpp` — full UART driver class (motion, read-back, homing, raw diag)
- `src/motion/MotionController.h/.cpp` — axis abstraction, velocity control, soft limits, homing state machine, NVS settings persistence
- `src/input/GamepadInput.h/.cpp` — Bluepad32 Xbox One S integration
- `src/cli/CLI.h/.cpp` — GNU-style serial CLI (help, move, stop, home, ping, diag, reset, save, estop, status, set ...)
- `include/config.h` — all pin assignments, gear ratios, motion defaults, task priorities
- `scripts/mks_setup.py` — standalone Python UART tool for direct driver config via FTDI

**Current gamepad mapping:**
- Left stick → pan/tilt velocity
- Right bumper (RB) → fine-speed mode (15% scale)
- Left bumper (LB) → latch laser on/off
- Left trigger → laser strobe ~25 Hz while held
- D-pad L/R → pan speed multiplier ±0.25 (range 0.25–10.0, default 4.25×)
- D-pad U/D → tilt speed multiplier ±0.25 (range 0.25–10.0, default 10.0×)
- A → stop all motion
- B → emergency stop (latching)
- Y → home both axes
- Menu → toggle soft limits

**Laser:** GPIO 13 → FET driver gate. Latch = steady on. Trigger = 25 Hz strobe (toggles every 20 ms task frame).

**Motion defaults (config.h):**
- Max speed: 180 deg/s at output shaft
- Accel: 30 deg/s²
- Fine speed scale: 0.15×
- Soft limits: DISABLED (re-enable after homing is configured)
- Pan limits: ±180°, Tilt limits: −45° to +90°

**NVS namespace:** `ptz_cfg` — persists across flashes. After changing config.h defaults, run `reset` then `save` from CLI.

### What Still Needs Doing

1. **MKS driver display config** — V3.6.1 CR_UART firmware blocks config commands via UART. Must configure these via physical OLED on each driver:
   - Work mode: CR_UART
   - Current, direction, microsteps (16)
   - ACC (menu item 18) — enable to get smooth ramps
   - 0_Mode = DirMode, 0_Speed = 2, 0_Dir = CW, then `Set 0` at desired home position

2. **Homing** — zero-mode must be set via display first (above). After that, `home all` CLI command or Y button should work. See `docs/MKS_Servo42C_Knowledge_Base.md`.

3. **32-byte pan response drain** — pan driver (V3.6.1) returns 32 bytes to 0x3A status query; leftover bytes may corrupt next read. Fix: add uart drain loop after each `doCommand` transaction.

4. **Soft limits** — set `SOFT_LIMITS_ENABLED true` in config.h once homing is operational so the controller knows the axis reference.

5. **Right trigger** — currently unassigned. Was formerly laser fire; reassigned to left trigger. Could be used for a second output or speed preset.

### How to Pick This Up Quickly

1. Open project in VSCode with PlatformIO extension
2. Read `include/config.h` — all pins and defaults are there
3. Read `docs/MKS_Servo42C_Knowledge_Base.md` — everything known about the drivers
4. Flash firmware, open serial monitor at 115200, type `help` for full CLI reference
5. Use `ping all` to verify UART to both drivers
6. Use `diag pan 3a` / `diag tilt 3a` to see raw driver responses
7. Use `scripts/mks_setup.py` with an FTDI adapter to configure drivers directly if needed
8. After any config.h default change: flash → `reset` → `save` in CLI
